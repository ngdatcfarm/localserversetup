## Bug 1: Bat Service - `AttributeError: 'float' object has no attribute 'isoformat'`

### Symptom
```
GET /api/bats/barns/barn-01 HTTP/1.1" 500 Internal Server Error
AttributeError: 'float' object has no attribute 'isoformat'
```

### Root Cause
File: `src/iot/bat_service.py`

- Line 151 (`_move`): `started_at` stored as `time.time()` (float/Unix timestamp)
- Line 44 (`list_by_barn`): Called `.isoformat()` assuming datetime object

```python
# Line 149-151 - WRONG
self._active_movements[bat_id] = {
    'direction': direction,
    'started_at': time.time(),  # <-- float, not datetime
    ...
}

# Line 44 - WRONG
bat['moving_started_at'] = active['started_at'].isoformat()  # <-- float has no .isoformat()
```

### Fix Applied
```python
# Line 44
bat['moving_started_at'] = datetime.fromtimestamp(active['started_at'], tz=timezone.utc).isoformat()
```

## Bug 3: Bats.js - Cannot Reverse Direction Mid-Motion

### Symptom
Đang đi xuống, bấm nút LÊN không có tác dụng (disabled)

### Root Cause
```javascript
// Line 355, 363
:disabled="isMoving(bat) || ..."
```

`isMoving(bat)` returns true for BOTH 'up' AND 'down' → cannot reverse without stopping first

### Fix Applied
```javascript
// LÊN button - disabled only when already going UP
:disabled="bat.moving_state === 'up' || !bat.device_id || loading[bat.id]"

// XUỐNG button - disabled only when already going DOWN
:disabled="bat.moving_state === 'down' || !bat.device_id || loading[bat.id]"
```

Now can reverse direction directly (backend already handles safety turn-off of opposite channel)

### To Enable Sync

**Step 1: Enable sync service**
```bash
curl -X POST https://192.168.1.9:8443/api/sync/config \
  -H "Content-Type: application/json" \
  -d '{"enabled": true}'
```

**Step 2: Get API token from cfarm.vn**
1. Login to https://cfarm.vn
2. Go to Settings / API
3. Copy API token

**Step 3: Set API token**
```bash
curl -X POST https://192.168.1.9:8443/api/sync/config \
  -H "Content-Type: application/json" \
  -d '{"api_token": "YOUR_TOKEN_HERE"}'
```

**Step 4: Trigger manual sync**
```bash
curl -X POST https://192.168.1.9:8443/api/sync/barns-devices-sync
```

### Issue: New Devices from MQTT Do Not Auto-Sync

When ESP32 sends heartbeat, device is auto-created in local DB via MQTT listener. However:
- No automatic trigger to sync new device to cloud
- Device needs manual sync via `/api/sync/barns-devices-sync`

**Solution Options:**
1. Enable periodic sync (every 60s via sync_interval)
2. Add webhook trigger when new device created
3. Manual sync after device creation
